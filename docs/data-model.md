# Data model

## In-memory game state (current)

Defined in [`backend/src/services/game/engine.ts`](../backend/src/services/game/engine.ts).

```ts
interface GameState {
  gameId: string;                  // 6-digit PIN
  hostToken: string;               // opaque, sent only to host on create_game
  hostSocketId: string | null;
  quiz: Quiz;                      // hard-coded in Phase 1; from DB in Phase 2
  status: 'lobby' | 'in_question' | 'between' | 'ended';
  currentQuestionIndex: number;
  questionStartedAt: number | null;   // epoch ms
  questionDeadlineAt: number | null;  // epoch ms
  players: Map<playerId, PlayerState>;
  answersByQuestion: Map<questionIndex, Map<playerId, AnswerRecord>>;
  createdAt: number;
}

interface PlayerState {
  playerId: string;     // server-issued UUID
  nickname: string;
  score: number;
  connected: boolean;
  joinedAt: number;
  socketId: string | null;
}

interface AnswerRecord {
  choice: number;       // 0-indexed
  submittedAt: number;
  scoreEarned: number;
}
```

**Persistence**: lives in process memory only. Wiped on backend restart (intentional for MVP — see [phases/deferred-redis.md](phases/deferred-redis.md) for the durable swap).

## Postgres schema (Phase 2)

Defined as Drizzle schema in `backend/src/db/schema.ts`. SQL below is for reference; migrations generated by `drizzle-kit`.

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  google_sub TEXT UNIQUE NOT NULL,
  email TEXT NOT NULL,
  name TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE quizzes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  questions JSONB NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX quizzes_owner_idx ON quizzes(owner_user_id);

CREATE TABLE games (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  quiz_id UUID NOT NULL REFERENCES quizzes(id),
  host_user_id UUID NOT NULL REFERENCES users(id),
  started_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  ended_at TIMESTAMPTZ,
  player_count INT NOT NULL DEFAULT 0,
  meeting_id VARCHAR(64)            -- reserved for Phase 4 (Google Meet)
);

CREATE TABLE game_results (
  game_id UUID NOT NULL REFERENCES games(id) ON DELETE CASCADE,
  player_nickname TEXT NOT NULL,
  final_score INT NOT NULL,
  final_rank INT NOT NULL,
  PRIMARY KEY (game_id, player_nickname)
);
```

### `quizzes.questions` JSONB shape

```json
[
  {
    "text": "What is 7 × 8?",
    "choices": ["54", "56", "64", "48"],
    "correct": 1,
    "limitMs": 15000
  }
]
```

Drizzle column definition: `questions: jsonb('questions').$type<Question[]>().notNull()`.

Validation rules (Phase 2 zod schema, applied at REST boundaries before insert/update):

- `title`: 1–80 chars
- `questions`: 1–50 items
- `text`: 1–300 chars
- `choices`: 2–4 strings, each 1–80 chars
- `correct`: integer ∈ `[0, choices.length)`
- `limitMs`: one of `[10000, 15000, 20000, 30000]`

## Ephemeral vs persistent

| Lives in memory only | Persisted to Postgres |
|---|---|
| Live game state (status, current question, timers) | Quiz definitions |
| Answer submissions during play | User accounts |
| Per-player connection state | Final scoreboard at `game_end` |
| Live leaderboard | Aggregate game metadata |
