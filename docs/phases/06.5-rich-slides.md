# Phase 6.5 — Rich Slides (media + info slides + slide editor)

**Status**: 🔜
**Depends on**: Phase 6 (question type system), Phase 7 (non-scored infrastructure)
**Goal**: Quizzes look and feel like Kahoot/Slides — each question is a "slide" with optional image, plus a new non-scored "info" slide type for context/intro screens. Editor is redesigned as a slide rail + main pane.

## Why this phase

Three features are deeply intertwined and pointless to ship separately: (a) per-question images, (b) info/description slides, (c) a slide-style editor. Doing them in one phase means one schema migration, one storage decision, one editor redesign — instead of three rounds of rework. Phase 6 must land first so the question type system exists; Phase 7 must land first because info slides reuse the non-scored infrastructure built there.

## Out of scope (intentional)

- Video / audio attachments (images only for v1)
- AI image generation in the AI question generator (text-only; image search is its own follow-up)
- Image cropping / filters in the editor (upload as-is)
- Asset library / reuse across quizzes (each upload is per-quiz)
- Drag-drop from desktop into the rail (file picker only for v1)
- Mobile editor — the slide rail collapses; full editing stays desktop-first

## Parallel PR strategy

The deliverables below are sequenced into **waves**. Every PR in the same wave touches a disjoint set of files and has no semantic dependency on the others — they can be developed and merged in any order.

```
Wave 1 (3 parallel PRs)     Wave 2 (4 parallel PRs)        Wave 3 (1 PR)
─────────────────────────   ─────────────────────────────   ──────────────
PR-A backend asset storage  PR-D image upload in editor    PR-H templates +
PR-B shared schema additions  └─ needs A + B                AI gen aware of
PR-C slide-rail editor shell  PR-E info slide editor + UI    images
                              └─ needs B + C
                            PR-F player image rendering
                              └─ needs B
                            PR-G player info-slide rendering
                              └─ needs B
```

Acceptance for the phase is "all PRs merged + smoke test passes". You can pick up any free PR slot at any time without blocking other developers/agents.

## Deliverables (one PR each)

### PR-A · Backend asset storage

**Wave**: 1 · **Depends on**: nothing

- [ ] Pick a backing store (recommend Cloudflare R2 — S3-compatible, $0 egress, free tier 10GB). Document in `decisions.md`.
- [ ] New env vars: `R2_ACCOUNT_ID`, `R2_ACCESS_KEY`, `R2_SECRET_KEY`, `R2_BUCKET`, `R2_PUBLIC_URL` (or equivalents)
- [ ] New endpoint `POST /assets/upload` — accepts `multipart/form-data`, validates MIME (`image/png`, `image/jpeg`, `image/webp`, `image/gif`), max 5MB, returns `{ assetId, url, width, height }`
- [ ] New `assets` table: `id`, `owner_user_id`, `mime`, `bytes`, `width`, `height`, `storage_key`, `public_url`, `created_at`
- [ ] Auth: only authenticated hosts can upload; rate-limit (10 req/min/user)
- [ ] Integration test that uploads a real PNG, verifies row + reachable URL

**Files**: `backend/src/routes/asset.routes.ts` (new), `backend/src/db/migrations/00XX_assets.sql` (new), `backend/src/db/repositories/assetRepo.ts` (new), `backend/src/config.ts` (modify), `.env.example` (modify), `docs/decisions.md` (modify)

### PR-B · Shared schema additions

**Wave**: 1 · **Depends on**: nothing

- [ ] Add optional `mediaRef?: { assetId: string; url: string; width: number; height: number }` to the question type in `shared/src/index.ts`
- [ ] Add new question type `'info'` to the question type union (text + optional `mediaRef`, no `choices`, no `correct`, no scoring)
- [ ] Update `scoreAnswer` to return 0 for `'info'` type (or skip it entirely upstream — pick one and document)
- [ ] Zod schema in `backend/src/routes/quiz.routes.ts` accepts both new shapes; reject `correct` on info type
- [ ] Backend migration: no DB change (questions are JSONB blobs already)
- [ ] Type checks pass; existing tests pass unchanged

**Files**: `shared/src/index.ts` (modify), `backend/src/routes/quiz.routes.ts` (modify validation schema only)

### PR-C · Slide-rail editor shell

**Wave**: 1 · **Depends on**: nothing (works against current question schema; image + info come later)

- [ ] Refactor `frontend/src/lib/components/QuizEditor.svelte` from vertical list to two-column layout:
  - Left rail: thumbnail per question (~140px wide), selected state, drag-to-reorder, "+" button at bottom
  - Main pane: only the currently-selected question, rendered at large size
- [ ] Keyboard nav: `↑/↓` move selection, `Cmd+D` duplicate slide, `Backspace` (with confirm) delete slide
- [ ] Selection state persists across save attempts; new slide is auto-selected after add/duplicate
- [ ] Empty state in rail when no questions; "Add question" CTA in main pane
- [ ] No schema changes; behavior identical to today for existing quizzes
- [ ] Mobile: rail collapses into a horizontal scroller above the main pane below 768px

**Files**: `frontend/src/lib/components/QuizEditor.svelte` (heavy refactor), `frontend/src/lib/components/SlideRail.svelte` (new), `frontend/src/lib/components/SlideEditor.svelte` (new — extract per-slide editing from current QuizEditor)

### PR-D · Image upload in the editor

**Wave**: 2 · **Depends on**: PR-A (upload endpoint), PR-B (mediaRef schema)

- [ ] Empty state: "+ Add image" button → opens file picker → uploads via `POST /assets/upload` → on success, sets `question.mediaRef`
- [ ] Filled state: image preview + "Replace" / "Remove" buttons
- [ ] Upload progress indicator; error states (too big, wrong type, network)
- [ ] Slide rail thumbnail uses the image as background when present

**Files**: `frontend/src/lib/components/SlideEditor.svelte` (modify), `frontend/src/lib/components/SlideRail.svelte` (modify thumbnail)

### PR-E · Info slide type — editor + creation UI

- [ ] "+ Add" button in the rail becomes a small menu: "Question" / "Info slide"
- [ ] `SlideEditor.svelte` renders a distinct UI when `question.type === 'info'`: large text editor, optional time-limit (default: host-advances), no choices, no correct-answer
- [ ] Rail thumbnail visually differentiates info slides (e.g. icon, different border)

### PR-F · Player image rendering

**Wave**: 2 · **Depends on**: PR-B

### PR-G · Player info-slide rendering

**Wave**: 2 · **Depends on**: PR-B

**Files**: `backend/src/services/game/lifecycle.ts` (modify), `backend/src/services/game/engine.ts` (modify), `frontend/src/routes/play/[gameId]/+page.svelte` (modify), `frontend/src/routes/host/[gameId]/+page.svelte` (modify)

### PR-H · Templates + AI generation awareness

**Wave**: 3 · **Depends on**: PR-B, PR-D, PR-G

- [ ] System templates ([backend/src/data/templates.ts](../../backend/src/data/templates.ts) or equivalent) can optionally include `mediaRef` and info slides
- [ ] At least one shipped template uses each new feature (smoke test for round-trip)
- [ ] AI generator (`POST /ai/generate-questions`) accepts optional `includeInfoSlides: boolean` param; when true, prompts the model to add 1–2 info slides at the start of the output

**Files**: `backend/src/data/templates.ts` (or repo), `backend/src/routes/ai.routes.ts` (modify), `backend/src/services/ai/prompts.ts` (or equivalent — modify)

## Verification (after all PRs merged)

1. Create a quiz: add 2 questions with images, 1 info slide between them, 2 regular questions. Save.
2. Host the quiz; play through with 2 player tabs. Confirm: images load, info slide displays without timer scoring, regular questions score correctly.
3. Upload limits: try a 6MB PNG (rejected), a `.exe` renamed to `.png` (rejected by MIME sniff or accepted depending on choice — document).
4. Reorder slides via the rail; reload; order persists.
5. Mobile (375×667): editor rail collapses, slides editable one at a time.
6. AI generate with `includeInfoSlides: true`; verify response contains at least one info slide.

## Acceptance criteria

- All 8 PRs merged
- Verification 1–6 pass
- No regression in existing quiz play (quizzes without images / info slides behave identically to today)
- Asset storage cost projection documented in `decisions.md` (e.g. R2 free tier supports ~50k average uploads)
